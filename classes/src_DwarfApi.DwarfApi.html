alloc

  • alloc(size: number, options?: MemoryAllocOptions): NativePointer

  • Allocates size bytes of memory on Frida's private heap, or, if size is a multiple of Process#pageSize, one or more raw memory pages managed by the OS. The allocated memory will be released when the returned NativePointer value gets garbage collected. This means you need to keep a reference to it while the pointer is being used by code outside the JavaScript runtime.

    Parameters

    • size: number

      Number of bytes to allocate.

    • Optional options: MemoryAllocOptions

      Options to customize the memory allocation.

    Returns NativePointer

    showData

    • showData(dataType?: string | DwarfDataDisplayType, dataIdentifier: string, userData: any, baseAddr?: number, thumbMode?: boolean): void

    • Shows CustomData in UI

      dataTypes allowed DwarfDataDisplayType or ['text', 'json', 'hex', 'disasm', 'sqlite3']

      Examples:

      showData('text', 'Test', 'Hello World');

      showData('hex', 'Bytes at eax', this.context['eax'].readByteArray(100));

      var baseAddr = ptr(this.context['eax']).toString();
      showData('diasm', 'Disasm from ' + baseAddr, this.context['eax'].readByteArray(100), baseAddr);
      showData('diasm', 'Disasm from ' + baseAddr, this.context['eax'].readByteArray(100), baseAddr, 'thumb');

      showData(2, 'Sample JSON', { name: 'Test', otherItem: 1 });

      showData('sqlite3', 'Sample Database', '/full/path/to/file.db'); //on Android it pulls db from Device

      Parameters

      • dataType: string | DwarfDataDisplayType = DwarfDataDisplayType.TEXT

        default TEXT

      • dataIdentifier: string

        some identifier string

      • userData: any

        your data to display for ['hex', 'disasm'] use ArrayBuffer

      • Optional baseAddr: number
      • Optional thumbMode: boolean

      Returns void
